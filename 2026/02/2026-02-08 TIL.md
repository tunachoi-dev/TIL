## 핵심 키워드 정리
- **데이터 vs 정보**: 날것의 기록(Data)을 가공해 얻은 유의미한 결과(Information)
- **DBMS:** 데이터를 체계적으로 관리하는 핵심 기반 시설
- **RDBMS**: 테이블(표) 간의 관계를 중심으로 구조화된 데이터베이스
- **PK & FK**: 데이터를 식별하는 '기본 키'와 데이터를 연결하는 '외래 키'
- **조건문**: 프로그램의 흐름을 제어하는 `if`, `else if`, `switch`, `삼항 연산자`

## SQL
### 1. 왜 파일 시스템 대신 DBMS인가?
단순히 파일(Excel, txt)에 데이터를 저장하면 다음과 같은 심각한 문제가 발생합니다.
- **데이터 중복 및 불일치**: 같은 데이터가 여러 곳에 저장되어 수정 시 데이터가 꼬임
- **동시성 제어 문제**: 여러 사용자가 동시에 접근할 때 데이터 일관성이 깨짐
- **무결성 및 보안**: 데이터의 정확성을 강제하기 어렵고, 세밀한 접근 권한 제어가 불가능함
### 2. 관계형 데이터베이스(RDBMS)의 핵심
RDBMS는 데이터를 **테이블(Table)** 단위로 관리하며 두 가지 키(Key)를 통해 무결성을 보장합니다.
- **기본 키(Primary Key, PK)**: 테이블 내에서 각 행을 유일하게 식별하는 주민등록번호 같은 역할 (중복 불가, NULL 불가)
- **외래 키(Foreign Key, FK)**: 다른 테이블의 PK를 참조하여 테이블 간의 '관계'를 맻어주는 연결 고리

## Java 조건문 (Conditional Statements)
### 1. if ~ else if ~ else
특정 조건이 참(`true`)일 때만 코드를 실행합니다.
- **독립 조건**: 여러 조건을 각각 체크할 때는 `if`문을 따로 사용합니다
- **연관 조건**: 여러 조건 중 하나만 실행하고 싶을 때는 `else if`로 묶어 효율성을 높입니다.
```java
if (price <= 10000) {
	discount += 1000;  // 1만원 이상 할인
}
if (age <= 10) {
	discount += 1000;  // 어린이 중복 할인 가능 (독립 조건)
}
```
### 2. switch문과 자바 14의 진화
단순히 값이 같은지 비교할 때는 `switch`문이 간결합니다.

|**구분**|**기존 switch**|**Java 14+ switch (추천)**|
|---|---|---|
|**문법**|`case 값: break;` 필수|`case 값 ->` 화살표 사용|
|**특징**|`break` 누락 시 다음 케이스까지 실행|코드 블록이 간결하고 직접 값을 반환 가능|
```java
// Java 14+ switch 예시
int coupon = switch (grade) {
	case 1 -> 1000;
	case 2 -> 2000;
	default -> 500;
}
```
### 3. 삼항 연산자 (?:)
단순히 조건에 따라 값을 대입할 때 유용합니다.
`String status = (age >= 18) ? "성인" : "미성년자";`
