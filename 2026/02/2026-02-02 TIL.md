# 2026-02-02 TIL
## 핵심 키워드 정리
### Java
- **변수(Variable):** 단 하나의 값을 저장할 수 있는 메모리 공간
- **자료형(Data Type):** 기본형(8개) & 참조형(주소값 저장)
- **상수(Constant) & 리터럴(Literal):** 변하지 않는 저장공간과 그 자체의 값
- **타입 추론(var):** 지역 변수의 타입 생략 기능
---
## 개념 설명
### 1. 변수의 선언과 초기화
변수는 물건을 담는 **'이름표가 붙은 상자'**
- **선언:** 어떤 종류의 물건을 담을지 결정하고 상자를 준비하는 것.
- **초기화:** 상자에 들어있을지도 모를 "쓰레기 값(Garbage Value)"을 치우고 처음으로 의미 있는 값을 넣는 것.
```java
int age;  // 선언: 정수(int)를 담을 age라는 상자 준비
age = 27;  // 초기화: 27라는 값을 저장 (대입 연산자 '='는 우측 값을 좌측에 넣음)
int year = 2027;  // 선언과 동시에 초기화
```
### 2. 기본형(Primitive) & 참조형(Reference)
자바의 데이터 타입은 크게 두 갈래로 나눔

|**분류**|**특징**|**종류**|
|---|---|---|
|**기본형**|실제 값(Data)을 메모리에 직접 저장|`boolean`, `char`, `byte`, `short`, `int`, `long`, `float`, `double`|
|**참조형**|값이 저장된 **메모리 주소**를 저장|기본형 8개를 제외한 모든 타입 (예: `String`, `Date`, 사용자 정의 클래스)|

실무에서는 정수는 `int`와 `long`, 실수는 `double`을 주로 사용. 특히 금융 관련 데이터를 다룰 때는 `long`의 범위를 잘 체크
### 3. 변수 명명 규칙 (Naming Convention)
식별자는 서로 구별될 수 있어야 하며, 협업을 위해 약속된 규칙을 따르는 것이 중료
- **필수 규칙:**
	- 대소문자 구분 (예: `apple`과 `Apple`은 다름)
	- 예약어 사용 불가 (예: `true`, `int` 등)
	- 숫자로 시작 불가
	- 특수문자는 `_`와 `$`만 허용
- **권장 관례 (Convention):**
	- **CamelCase:** 변수와 메서드 이름 (예: `userAge` `totalPrice`)
	- **PascalCase:** 클래스 이름 (예: `VariableEx`, `OrderController`)
	- **의미 있는 이름:** `int a = 10;` 보다는 `int maxLoginAttempts = 10;` 처럼 용도를 알 수 있게 작성
### 4. 문자열(String) 연산의 특이점
`String`은 참조형이지만 기본형처럼 `+` 연산자로 결합 가능.
**결합 방향 주의:** 연산은 왼쪽에서 오른쪽으로 진행
```java
System.out.println(7 + 7 + "");  // "14" (숫자 합산 후 문자열 결합)
System.out.println("" + 7 + 7);  // "77" (문자열 결합 후 문자열 결합)
```
### 5. 상수(Constant)와 리터럴(Literal)
- **상수:** 한 번 저장하면 바꿀 수 없는 공간. `final` 키워드를 사용하며, 보통 대문자로 명명
- **리터럴:** `100`, `3.14`, `'A'`와 같이 값 그 자체를 의미
```java
final int MAX_SPEED = 100;  // 상수
// MAX_SPEED = 120;  // 컴파일 에러 발생
```
### 6. 타입 추론 (var)
JDK 10부터는 지역 변수에 한해 `var` 키워드를 사용 가능. 컴파일러가 대입되는 리터럴을 보고 타입을 자동으로 결정
```java
var name = "tunachoi-dev"  // String으로 추론
var age = 27;  // int로 추론
```
