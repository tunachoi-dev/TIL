# 핵심 키워드 정리
- **Java.lang 패키지**: 자바 언어의 가장 기본 클래스들(`Object`, `String`, `System` 등)을 담은 패키지로, 모든 클래스에 **자동으로 import**됨.
- **Object 클래스**: 자바 내 모든 클래스의 **최상위 부모 클래스**. 부모가 없는 클래스는 컴파일러가 묵시적으로 `extends Object`를 추가.
- **다형적 참조**: `Object` 타입은 모든 자바 객체를 참조할 수 있는 능력을 가짐.
- **동일성(Identity)** `==` 연산자를 사용하여 두 객체의 **메모리 주소(참조값)** 가 같은지 확인하는 것.
- **동등성(Equality)**: `equals()` 메서드를 사용하여 두 객체가 **논리적으로 같은 가치**를 지니고 있는지 확인하는 것.
- **toString()**: 객체의 정보를 문자열로 반환하는 메서드로, 기본값은 `클래스명@해쉬코드`

# 핵심 내용 정리
## Java
### 1. Object 클래스가 최상위 부모인 이유
- **공통 기능 제공**: 객체 정보 출력(`toString`), 객체 비교(`equals`), 클래스 정보 확인(`getClass`) 등 모든 객체에 필요한 기능을 표준화하여 제공.
- **다형성의 기본 구현**: 어떤 타입의 객체든 `Object` 타입으로 받을 수 있어, 타입에 관계없이 객체를 통합 관리(예: `Object[]`)할 수 있다.
### 2. Object 다형성의 한계와 해결
- **한계**: `Object`는 모든 객체를 참조할 수 있지만, 자식 클래스만이 가진 고유한 메서드(예: `Dog.sound()`)는 직접 호출할 수 없습니다.
### 3. toString() 오버라이딩과 OCP
- **오버라이딩**: 기존 참조값 대신 객체의 상태(필드 데이터)를 출력하도록 재정의하면 디버깅과 로깅이 비약적으로 편리해집니다.
- **OCP(개방-폐쇄 원칙) 준수**: `System.out.println()`이나 별도의 프린터 클래스가 구체적인 타입(Car, Dog)이 아닌 추상적인 `Object`와 `toString()`에 의존하게 설계하면, 새로운 클래스가 추가되어도 출력 로직을 수정할 필요가 없습니다.
### 4. 동일성(Identity) vs 동등성(Equality)
| 구분      | 비교 방식          | 의미        | 기준                         |
| ------- | -------------- | --------- | -------------------------- |
| **동일성** | `==` 연산자       | 물리적으로 같은가 | 메모리 주소(참조값)                |
| **동등성** | `equals()` 메서드 | 논리적으로 같은가 | 클래스에서 정의한 로직<br>(ID, 번호 등) |
**주의**: `Object`가 기본으로 제공하는 `equals()`는 내부적으로 `==` 비교를 수행. 따라서 논리적 비교가 필요한 경우 반드시 클래스에서 `equals()`를 **재정의(Override)** 해야 함.
