# 핵심 키워드 정리
- **접근 제어자**: 외부에서 특정 필드나 메서드에 접근하는 것을 허용하거나 제한하는 기능
- **private**: 모든 외부 호출을 막고 클래스 내부에서만 사용
- **default**: 같은 패키지 안에서만 호출을 허용
- **protected**: 같은 패키지 안에서, 그리고 상속 관계의 호출을 허용
- **public**: 모든 외부 호출을 허용
- **캡슐화**: 데이터와 기능을 하나로 묶고, 외부에 꼭 필요한 기능만 노출해 나머지는 숨기는 개념

# Java
## 1. 접근 제어자(Access Modifier)의 정의와 목적
- **정의**: 클래스, 필드, 생성자, 메서드에 대한 외부의 접근을 허용하거나 제한하기 위해 사용하는 키워드
- **목적**: 의도치 않은 데이터 변조를 막고, 객체의 내부 로직을 보호하기 위해 클래스 외부에서의 접근을 통제하는 역할
## 2. 접근 제어자의 4가지 종류 및 허용 범위
접근을 차단하는 강도가 높은 순서대로 다음과 같이 분류된다.
- **private**: 모든 외부 호출을 차단하며, 오직 해당 요소가 선언된 **클래스 내부**에서만 접근할 수 있다.
- **default (package-private)**: **동일한 패키지 내**에서만 접근을 허용한다. (접근 제어자를 명시하지 않을 경우 기본으로 적용)
- **protected**: 동일한 패키지 내에서 접근 가능하며, 패키지가 다르더라도 해당 클래스를 **상속받은 자식 클래스**에서의 접근을 허용한다.
- **public**: 어떠한 제한 없이 **모든 외부 호출**을 허용한다.
### 클래스 레벨의 제어자 적용 규칙
- **정의**: 객체의 속성(데이터)과 기능(메서드)을 하나로 묶고, 외부에 꼭 필요한 기능만 노출한 채 내부의 데이터와 구현 디테일을 숨기는(정보 은닉) 객체지향 프로그래밍의 핵심 개념.
- **원칙 1 (데이터 은닉)**: 객체의 상태를 나타내는 데이터(필드)는 외부에서 직접 접근할 수 없도록 원칙적으로 `private`으로 제한해야 한다. 데이터에 대한 접근과 수정은 오직 해당 객체가 제공하는 검증된 메서드를 통해서만 이루어지도록 통제해야 한다.
- **원칙 2 (기능 은닉)**: 외부 객체와의 상호작용에 반드시 필요한 인터페이스(메서드)만 `public`으로 열어둔다. 데이터 검증이나 내부 연산 등 외부에서 알 필요가 없는 내부 로직은 `private`으로 숨겨, 객체 사용의 복잡도를 낮추고 외부 결합도를 줄여야 한다.
