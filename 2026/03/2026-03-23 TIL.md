# 핵심 키워드 정리
- **공유 참조**: 여러 변수가 하나의 객체 지향 인스턴스를 동시에 참조하는 상태
- **사이드 이펙트(Side Effect)**: 주된 목적 외에 의도치 않게 다른 변수의 값이나 상태를 변경시키는 부작용을 의미
- **불변 객체(Immutable Object)**: 객체 생성 이후 내부의 상태(필드 값)를 절대 변경할 수 없는 객체
- **String 풀(String Pool)**: 문자열 리터럴을 재사용하여 메모리 효율을 높이기 위해 힙 영역 내에 구축된 저장 공간.
- **StringBuilder**: 가변(Mutable) 문자열 클래스로, 문자열을 자주 변경해야 할 때 성능 최적화를 위해 사용
- **메서드 체이닝(Method Chaining)**: 메서드가 자기 자신의 참조를 반환하여 연속적으로 메서드를 호출하는 기법

# Java
## 1. 불변 객체 (Immutable Object)
- **필요성**: 참조값 공유를 통해 발생하는 사이드 이펙트는 컴파일 단게에서 잡아낼 수 없는 런타임 오류의 주원인이 됨.
- **구현 방법**: 필드를 `final`로 선언하고 `setter`를 제공하지 않아 상태 변경을 원천 차단
- **값 변경 시**: 불변 객체의 값을 변경하려면 기존 객체를 수정하는 것이 아니라, 변경된 값을 가진 **새로운 객체를 생성하여 반환**해야 함.
## 2. String 클래스의 기본 특성
- **불변성**: `String`은 내부적으로 `final char[]` (또는 `byte[]`)를 사용하며, 한 번 생성되면 그 내용을 바꿀 수 없는 불변 객체
- **리터럴과 풀**: `String str = "Hello"`와 같이 리터럴을 사용하면 String 풀에서 기존에 같은 문자열이 있는지 확인하고 재사용하여 메모리를 절약
- **비교 주의점**: 문자열 비교 시 `==`은 참조값을 비교하므로, 반드시 내용물을 비교하는 `equals()` 메서드를 사용해야 함.
## 3.StringBuilder와 성능 최적화
- **String의 한계**: 불변 객체인 `String`끼리 더하기 연산을 수행할 때마다 계속해서 새로운 객체가 생성되어 CPU와 메모리 소모가 커짐.
- **StringBuilder 활용**: 내부에 가변 배열을 가지고 있어 루프 안에서 문자열을 합치는 등 빈번한 수정이 일어날 때 성능상 압도적으로 유리
- **최종 변환**: 문자열 조작이 끝나면 `toString()`을 호출하여 다시 안전한 불변 객체인 `String`으로 변환하여 사용하는 것이 권장.
## 4. String 클래스의 주요 메서드
- **정보 조회**: `length()`, `isEmpty()`, `isBlank()`, `charAt(index)` 등이 있음.
- **비교 및 검색**: `contains()`, `startsWith()`, `endsWith()`, `indexOf()` 등을 제공
- **변환 및 조작**: `substring()`, `replace()`, `toLowerCase()`, `trim()`, `split()`, `join()` 등을 활용할 수 있음.
