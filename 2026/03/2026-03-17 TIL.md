# 핵심 키워드 정리
- **상속 (Inheritance)**: 기존 클래스의 필드와 메서드를 새로운 클래스에서 재사용하게 해주는 객체 지향의 핵심 요소.
- **extends**: 상속을 구현하기 위해 사용하는 키워드로, 대상은 단 하나만 선택할 수 있는 단일 상속을 원칙으로 함.
- **메서드 오버라이딩 (Method Overriding)**: 부모로부터 물려받은 기능을 자식 클래스에서 재정의하는 것.
- **super**: 부모 클래스의 참조를 나타내며, 부모의 필드, 메서드, 생성자를 호출할 때 사용.
- **다형성 (Polymorphism)**: 한 객체가 여러 타입의 객체로 취급될 수 있는 능력.
- **업캐스팅 & 다운캐스팅**: 참조 타입을 부모 타입으로 바꾸는 것(업캐스팅)과 자식 타입으로 바꾸는 것(다운캐스팅).
- **instanceof**: 참조 변수가 어떤 인스턴스를 참조하고 있는지 확인하기 위해 사용하는 키워드.

# Java
## 1. 상속의 기본과 단일 상속
- **상속 관계**: `extends` 키워드를 사용하여 부모 클래스 (슈퍼 클래스)의 기능을 자식 클래스(서브 클래스)가 물려받는 관계이다.
- **단일 상속의 이유**: 자바는 다중 상속을 지원하지 않는다. 만약 여러 부모를 상속받으면 어떤 부모의 메서드를 호출해야 할지 모호해지는 '다이아몬드 문제'가 발생하기 때문이다.
## 2. 상속과 메모리 구조
- **인스턴스 생성**: 자식 클래스를 생성하면 메모리(힙 영역)에는 부모와 자식 클래스 정보가 모두 포함된 하나의 참조값(`X001`) 안에 각각의 공간이 구분되어 생성된다.
- **메서드 호출 우선순위**: 호출하는 변수의 타입을 기준으로 선택하며, 만약 현재 타입에서 기능을 찾지 못하면 상위 부모 타입으로 올라가서 기능을 찾는다.
## 3. super 키워드와 생성자 규칙
- **부모 참조**: 자식과 부모의 필드/메서드 이름이 같을 때 `super`를 통해 부모의 것을 명시적으로 호출할 수 있다.
- **생성자 필수 호출**: 상속 관계에서 자식 클래스의 생성자 첫 줄에는 반드시 `super(...)`를 통해 부모 생성자를 호출해야 한다. 부모의 기본 생성자인 경우 생략 시 컴파일러가 자동으로 추가해 준다.
## 4. 다형적 참조와 그 한계
- **부모는 자식을 품는다**: `Parent poly = new Child()`처럼 부모 타입 변수는 자식 인스턴스를 참조할 수 있다.
- **접근의 한계**: 부모 타입으로 선언된 변수는 인스턴스 내부에 자식이 있더라도 부모 클래스의 기능만 기본적으로 알고 있기 때문에, 자식 클래스 전용 메서드는 호출할 수 없다.
## 5. 캐스팅 (Casting)
- **업캐스팅**: 자식 타입을 부모 타입에 대입하는 것으로, 항상 안전하며 명시적 기입을 생략할 수 있다.
- **다운캐스팅**: 부모 타입을 자식 타입으로 변환하는 것이다. 인스턴스에 존재하지 않는 타입으로 변환을 시도하면 `ClassCastException` 런타임 오류가 발생하므로 주의해야 한다.
- **instanceof 활용**: 다운캐스팅 전에는 반드시 `instanceof`로 타입을 확인하는 것이 안전하다. 자바 16부터는 `if (parent instanceof Child child)`와 같이 체크와 동시에 변수 선언이 가능하다.
## 메서드 오버라이딩
- **절대 우선권**: 다형적 참조 상황(`Parent poly = new Child()`)에서 메서드를 호출할 때, 만약 자식 클래스에 오버라이딩 된 메서드가 있다면 **오버라이딩 된 메서드가 항상 우선적으로 실행**된다.
- **멤버 변수와의 차이**: 메서드는 오버라이딩 되어 우선권을 가지지만, 멤버 변수는 오버라이딩 되지 않고 변수 타입에 따라 호출된다.
