# 핵심 키워드 정리
- **메서드 영역 (Method Area)**: 클래스 정보, `static` 변수, 런타임 상수 풀이 보관되는 프로그램 전역 공유 메모리 공간.
- **스택 영역 (Stack Area)**: 메서드가 호출될 때마다 스택 프레임이 생성되어 지역 변수와 매개변수가 저장되는 공간 (후입선출 구조).
- **힙 영역 (Heap Area)**: `new` 키워드로 생성된 객체(인스턴스)와 배열이 보관되는 공간이며 가비지 컬렉션(GC)의 관리 대상.
- **인스턴스 변수/메서드**: `static`이 붙지 않은 멤버로 인스턴스를 생성해야만 힙 영역에 할당되거나 호출할 수 있는 객체 소속의 데이터와 기능
- **클래스 변수/메서드 (`static`)**: 클래스가 로드될 때 메서드 영역에 생성되며, 인스턴스 생성 없이 클래스명을 통해 공용으로 접근할 수 있는 데이터와 기능.
- **final 키워드**: 변수에 최초 한 번만 값을 할당할 수 있게 하며, 어떠한 변경도 불가능하게 만드는 제약 키워드
- **상수 (Constant)**: `static final` 키워드를 사용하여 선언하며, 프로그램 전체에서 변하지 않고 단 하나만 존재하는 고정된 값.
- **final 참조형 변수**: 할당된 참조값(메모리 주소) 자체를 다른 객체로 변경하는 것은 불가능하지만, 참조하고 있는 대상 객체 내부의 값은 변경할 수 있음.

# Java
## 1. 자바 메모리 구조
자바가 실행될 때 사용하는 메모리 공간은 크게 세 가지로 나뉜다.
- **메서드 영역 (Method Area)**: 프로그램을 실행하는 데 필요한 공통 데이터를 관리하는 전역 공간. 클래스의 실행 코드(바이트 코드), 메서드 코드, `static` 변수, 런타임 상수 풀 등이 이 영역에 보관된다.
- **스택 영역 (Stack Area)**: 실제 메서드가 호출되고 실행되는 영역으로, 후입선출(LIFO) 구조를 가진다. 메서드가 호출될 때마다 '스택 프레임'이 생성되어 지역 변수(매개변수 포함)를 보관하며, 메서드가 종료되면 스택 프레임과 함께 해당 지역 변수들도 메모리에서 제거된다.
- **힙 영역 (Heap Area)**: `new` 키워드를 통해 생성된 객체(인스턴스)와 배열이 존재하는 공간이다. 더 이상 참조되지 않는 객체는 가비지 컬렉터(GC)에 의해 자동으로 제거된다.
## 2. static 변수 (클래스 변수)
- **정의**: 멤버 변수에 `static` 키워드를 붙인 것으로, 인스턴스마다 독립적으로 생성되는 것이 아니라 클래스에 소속되어 하나만 생성되는 공용 변수이다.
- **생명 주기**: JVM이 해당 클래스를 로딩하는 순간 메서드 영역에 생성되며, 프로그램이 종료될 때까지 유지되므로 가장 긴 생명 주기를 가진다.
- **접근 방법**: 인스턴스 생성을 통하지 않고 `클래스명.변수명` 형식으로 직접 접근하는 것이 원칙이다.
## 3. static 메서드 (클래스 메서드)
- **정의**: 메서드에 `static` 키워드를 붙인 것으로, 객체 생성 없이 클래스명을 통해 바로 호출할 수 있는 공용 메서드이다.
- **사용 제약 (중요)**: `static` 메서드 내부에서는 오직 `static` 변수나 `static` 메서드만 호출할 수 있다. 인스턴스 생성 없이 호출되므로, 참조값(`this`)이 필요한 인스턴스 변수나 인스턴스 메서드에는 직접 접근할 수 없다.
## 4. final 키워드의 정의와 특징
- **정의**: 변수에 `final` 키워드가 붙으면, 해당 변수는 최초 한 번만 값을 할당할 수 있으며 이후에는 어떠한 경우에도 값을 변경할 수 없다.
- **적용 위치에 따른 규칙**:
	- **지역 변수**: 선언 시점이나 이후에 최초 한 번만 값을 할당할 수 있다.
	- **매개변수**: 메서드 호출 시점에 넘어온 값을 메서드 내부에서 임의로 변경할 수 없다.
	- **멤버 변수(필드)**: 필드 선언 시점에 직접 초기화하거나, 생성자를 통해서 반드시 한 번 초기화해야 한다.
## 5. 상수 (Constant)
- **정의**: 애플리케이션 전반에서 변하지 않고 단 하나만 존재하는 고정된 값을 의미한다.
- **구현**: 자바에서 `static final` 키워드를 조합하여 상수를 선언한다.
- **특징 및 관례**:
	- 관례적으로 변수명을 대문자로 작성하며, 단어의 구분은 언더스코어(`_`)를 사용한다 (예: `MAX_USERS`).
	- 매직 넘버(의미를 알 수 없는 숫자)의 사용을 방지하고, 값을 중앙에서 한 번에 관리할 수 있어 유지보수성이 극대화된다.
## 6. final과 참조형 변수 (주의점)
- 참조형 변수에 `final` 키워드를 사용할 경우, 해당 변수가 가리키는 **참조값(메모리 주소) 자체를 다른 객체로 변경하는 것만 불가능**하다.
- 참조값 자체는 고정되지만, 참조하고 있는 대상 **객체 내부의 값(인스턴스 변수)은 `final`로 선언되지 않았다면 얼마든지 변경할 수 있다.**
