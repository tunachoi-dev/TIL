# 핵심 키워드 정리
- **래퍼 클래스 (Wrapper Class)**: 기본 타입(int, double 등)을 객체로 감싸서 사용할 수 있게 해주는 클래스
- **박싱(Boxing) & 언박싱(Unboxing)**: 기본 타입을 래퍼 클래스로 변환하는 것을 박싱, 반대로 객체에서 기본 타입을 꺼내는 것을 언박싱이라 한다.
- **오토 박싱(Auto-Boxing)**: 자바 5부터 도입되어 컴파일러가 개발자 대신 박싱과 언박싱을 자동으로 처리해 주는 기능
- **Class 클래스**: 자바에서 실행 중인 애플리케이션의 클래스와 인터페이스의 메타데이터를 관리하는 클래스
- **리플렉션 (Reflection)**: 클래스에 선언된 메서드, 필드 등의 정보를 실시간으로 조작하는 기술
- **열거형(Enum Type)**: 미리 정의된 상수들의 집합을 정의할 수 있는 특별한 클래스 타입

# Java
## 1. 래퍼 클래스와 오토 박싱
- **객체화의 이유**: 기본 타입은 객체가 아니므로 메서드를 가질 수 없고 컬렉션 프레임워크에 담을 수 없는 등의 한계가 있어 래퍼 클래스가 필요합니다.
- **불변 객체**: 모든 래퍼 클래스는 내부 값을 변경할 수 없는 불변 객체입니다.
- **오토 박싱의 편의성**: `Integer box = 10;` 처럼 작성하면 컴파일러가 `Integer.valueOf(10)`으로 자동 변환해 주어 기본 타입과 객체 간 변환이 매우 편리합니다.
- **성능 고려**: 기본 타입은 메모리 효율이 좋지만, 래퍼 클래스는 객체 생성 비용이 추가되므로 연산이 매우 많은 경우 기본 타입을 우선 고려해야 합니다.
## 2. Class 클래스와 메타데이터
- **역할**: 클래스의 이름, 부모 클래스, 인터페이스, 메서드, 필드 정보 등을 모두 담고 있습니다.
- **조회 방법**: `String.class` (클래스 직접 조회), `obj.getClass()` (인스턴스에서 조회), `Class.forName("문자열")` (이름으로 조회) 방식이 있음
- **인스턴스 생성**: `getDeclaredConstructor().newInstance()`를 통해 클래스 정보만으로 동적으로 객체를 생성할 수 있음
## 3. Enum(열거형)의 도입 배경과 장점
- **문자열/정수 상수의 한계**: 타입 안전성이 부족하고 오타로 인한 런타임 오류가 발생하기 쉽다
- **타입 안전성**: Enum은 정의된 상수 외의 다른 값은 컴파일 시점에 차단하여 코드의 신뢰성을 높임
- **주요 메서드**: `values()` (모든 상수 배열 반환), `name()` (상수 이름 문자열 반환), `ordinal()` (선언 순서 반환) 등
    - 주의: `ordinal()`은 순서 변경 시 치명적인 버그를 유발할 수 있으므로 가급적 사용을 자제

## 4. 열거형의 확장 (Enum과 멤버 변수)
- **객체 지향적 활용**: Enum은 클래스이므로 필드(멤버 변수)와 생성자, 메서드를 가질 수 있음
- **캡슐화**: 상수마다 서로 다른 데이터를 할당하고 관련 로직을 Enum 내부에 포함시켜 데이터와 로직을 한곳에서 관리
