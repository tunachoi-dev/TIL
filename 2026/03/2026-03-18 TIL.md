# 핵심 키워드 정리
- **추상 클래스 (Abstract Class)**: 실제 인스턴스가 존재하지 않는 추상적인 개념의 클래스로, 상속을 목적으로 하며 `abstract` 키워드를 사용함.
- **추상 메서드 (Abstract Method)**: 부모 클래스에 정의는 되어 있으나 바디(`{}`)가 없는 메서드로, 자식 클래스가 반드시 오버라이딩해야 함.
- **순수 추상 클래스**: 모든 메서드가 추상 메서드인 클래스로, 실행 로직 없이 다형성을 위한 설계도(껍데기) 역할만 수행함.
- **인터페이스 (Interface)**: 순수 추상 클래스를 더 편리하게 사용할 수 있게 만든 기능으로, `interface` 키워드를 사용하며 다중 구현을 지원함.
- **다중 구현 (Multiple Implementation)**: 자바의 클래스 상속과 달리 한 클래스가 여러 개의 인터페이스를 동시에 구현할 수 있는 것.
- **다이아몬드 문제 (Diamond Problem)**: 다중 상속 시 어떤 부모의 메서드를 호출해야 할지 모호해지는 문제로, 인터페이스는 구현부가 없어 이 문제를 해결함.

# Java
## 1. 다형성의 필요성
- **타입의 일치**: 서로 다른 클래스(예: Dog, Cat 등)를 하나의 배열에 담거나 공통 메서드의 매개변수로 사용하려면 공통된 부모 타입(예: Animal)이 필요함
- **코드의 재사용**: 부모 타입인(예: Animal)을 참조하게 하면, 새로운 동물이 추가되어도 사용하는 쪽의 코드를 변경하지 않고 그대로 재사용할 수 있음.
## 2. 추상 클래스와 추상 메서드
- **제약의 보완**: 일반 부모 클래스는 실수로 인스턴스를 생성하거나, 자식에서 메서드 오버라이딩을 누락할 위험이 있음.
- **직접 생성 불가**: `abstract class`는 인스턴스를 직접 생성할 수 없어 추상적인 개념이 실체화되는 실수를 방지함
- **강제 구현**: `abstract method`가 하나라도 있다면 해당 클래스는 반드시 추상 클래스로 선언해야 하며, 자식은 이 메서드를 반드시 구현해야 함.
## 3. 인터페이스의 특징
- **규격과 설계도**: 인터페이스는 모든 메서드가 `public abstract`이며, 이를 구현하는 클래스가 지켜야 할 규격(규약)을 제공함.
- **생략 권장**: 메서드의 `public abstract`와 멤버 변수의 `public static final` 키워드는 생략이 권장됨.
- **구현 키워드**: 상속(`extends`) 대신 구현(`implements`) 키워드를 사용하며, UML에서는 점선 화살표로 표기함
## 4. 인터페이스와 다중 구현
- **다중 상속 금지**: 클래스는 메서드 실행문이 중복되는 '다이아몬드 문제' 때문에 단일 상속만 허용함.
- **다중 구현 허용**: 인터페이스는 메서드 바디가 없으므로, 여러 인터페이스에서 동일한 이름의 메서드가 넘어와도 어차피 자식 클래스에서 하나로 구현하기 때문에 충돌 문제가 없음
## 5. 클래스 상속과 인터페이스 구현 동시 사용
- **상속과 구현**: 하나의 클래스는 부모 클래스를 상속받으면서 동시에 여러 인터페이스를 구현할 수 있음
- **선언 순서**: `extends`를 먼저 선언하고 그 뒤에 `implements`를 작성해야 함.
